وث᱆يقة المتطلبات الربمج᱆ية التفص᱆يل᱆ية لمنصة Matrix / AllInAll ERP
1. تعر᱅يف النظام
نريد بناء منصة برمجية باسم Matrix، والمنتج األسايس داخلها هو AllInAll ERP. الهدف ليس مجرد موقع تعريفي، بل نظام كامل إلدارة دورة العميل من
أول طلب الخدمة، مرورًا بالمراجعة والعرض والسداد والتنفيذ، مث تشغيل نظام ERP للشركات.
النظام يتكون من أربع طبقات رئيسية:
الموقع العام Public Website.
لوحة التحكم الداخلية Matrix Control Center.
نظام إدارة العمالء والطلبات والعروض والتنفيذ.
نظام ERP متعدد الشركات والفروع.
يجب أن يدعم النظام العربية واإلنجلزيية من البداية، مع RTL/LTR، ووضع داكن وفاتح، وأن يكون قابال ً للتشغيل كـ SaaS أو كنسخة منفصلة للعميل.
2. التقن᱆ية المقرتحة
Frontend
Next.js App Router.
React.
TypeScript.
CSS Modules أو Tailwind حسب قرار الفريق.
دعم كامل للعربية RTL واإلنجلزيية LTR.
Forms مبنية بشكل type-safe.
Client validation + Server validation.
Backend
NestJS أو Next.js API في البداية، مث فصل API مستقل عند التوسع.
TypeScript.
Prisma ORM.
PostgreSQL.
Redis ،للكاش rate limiting، queues الحقًا.
JWT/Auth sessions.
REST API كبداية، مع قابلية GraphQL الحقًا إن لزم.
Infrastructure
Ubuntu Server.
Nginx Reverse Proxy.
PM2 أو Docker Compose.
SSL via Certbot.
