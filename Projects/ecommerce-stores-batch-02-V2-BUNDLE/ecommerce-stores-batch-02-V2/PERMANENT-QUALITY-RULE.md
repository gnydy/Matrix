# Permanent Quality Rule for the 1200 Templates

This rule is locked for all upcoming batches:

1. No batch is accepted if differences are only color, text, logo, or product names.
2. Every template must have a different concept, layout, visual rhythm, navigation feel, UX flow, and sales positioning.
3. Every ecommerce template must contain full frontend commerce experience: home, listing, product details, cart, checkout mock, wishlist/account logic or equivalent, filters, order summary, and customer account mock.
4. System templates must include realistic dashboard states, tables, forms, stats, settings, empty/loading/error states, and role-based UI concept.
5. Portfolio display must show the UX pattern and the reason the template is commercially sellable.

The old mistake is explicitly rejected: recolor-only templates are not portfolio-ready.
